# Инструкция: DEMO-РЕЖИМ (CoinLover)

Этот документ описывает логику работы и доступа к Demo-режиму приложения.

## 🛡 Безопасность и Приватность

По умолчанию приложение CoinLover теперь всегда открывается в **Demo-режиме**.
Это сделано для того, чтобы ваши реальные финансовые данные были скрыты от случайных посетителей.

## 🔗 Способы входа

### 1. Публичный доступ (Demo)

* **Ссылка:** Любая стандартная ссылка на приложение (например, `coinlover.app`)
* **Поведение:** Если пользователь заходит впервые или не авторизован через секретный ключ, он попадает в **Demo-режим** (подключаются таблицы `Transactions-demo` и `Configs-demo`).

### 2. Секретный вход (Real Mode)

Чтобы приложение переключилось на ваши реальные данные, необходимо использовать один из секретных параметров в URL. Приложение запомнит ваш выбор в `localStorage` браузера.

* `?demo=real`
* `?mode=real`
* `?key=99`
* `#real` (якорь в конце URL)

**Пример:** `https://coinlover.app/?key=99`

### 3. Ручное переключение (Long-Press)

Внутри приложения можно быстро переключиться между режимами, используя скрытый жест:

1. Найдите иконку **CircleDollarSign** (зеленая монетка) в верхнем левом углу.
2. Удерживайте палец (или кнопку мыши) на ней более **0.7 секунды**.
3. Приложение завибрирует и перезагрузится в противоположном режиме.

## 🎨 Визуальная индикация

* В **Demo-режиме** рядом с иконкой монетки отображается стильный пульсирующий бейдж **Demo**.
* В реальном режиме бейдж отсутствует.

## 🛠 Техническая реализация

* **LocalStorage:** Ключ `coinlover_demo`. Значение `"false"` — реальный режим, любое другое значение (или отсутствие ключа) — Demo.
* **Google Sheets:** При обнаружении флага Demo сервис `googleSheetsService` автоматически добавляет параметр `&demo=true` к запросам к Google Apps Script.
* **Авто-очистка URL:** После активации режима через URL, приложение автоматически очищает адресную строку от секретных параметров для сохранения "чистого" внешнего вида.
