# CoinLover: Технический дизайн и концепция UI/UX (2025-2026)

## 1. Анализ рынка и тренды 2025-2026

Современные приложения для личных финансов отходят от образа сложных бухгалтерских таблиц (какими были ранние версии CoinKeeper) в сторону «интеллектуальных финансовых партнеров». Главная цель — снижение когнитивной нагрузки и финансовой тревожности пользователя.

**Ключевые тренды:**

* **Bento Grid Layout:** Использование модульной сетки для организации виджетов. Это позволяет пользователю считывать главную информацию (баланс, расходы за день) за секунду без визуального шума.
* **Linear Style & Glassmorphism:** Глубокие темные интерфейсы с эффектом матового стекла. Создает ощущение премиального, технологичного, но в то же время спокойного продукта.
* **Hyper-Personalization & AI-ready:** Интерфейс должен быть готов к предиктивным подсказкам и адаптации под привычки (например, быстрый ввод частых категорий).
* **Micro-interactions:** Плавные анимации и мгновенный визуальный отклик на любое действие (особенно при вводе трат), что повышает доверие к системе.
* **Zero UI / One-tap entry:** Максимальное упрощение ввода транзакций. Пользователь не должен продираться через 5 экранов, чтобы записать покупку кофе.

## 2. Концепция CoinLover (MVP)

**Философия:** Только самое необходимое. Никаких сложных интеграций с банками, управления долгами или сложного бюджетирования на первых порах. Фокус на осознанности трат и простоте ввода.

### Основные сущности

1. **Счета (Кошельки):** Откуда уходят деньги (Наличные, Карта).
2. **Категории:** Куда уходят деньги (Продукты, Транспорт, Кафе).
3. **Транзакции:** Доходы и расходы.

### Структура приложения (Необходимый минимум)

1. **Главный экран (Dashboard - Bento Grid):**
    * *Виджет "Общий баланс"*: Крупно, четко.
    * *Виджет "Быстрый ввод"*: Главная кнопка или область для мгновенного добавления расхода (в идеале — drag-and-drop монетки в категорию, как в CoinKeeper, но в современном исполнении, либо умное поле ввода).
    * *Виджет "Траты за сегодня/месяц"*: Прогресс-бар или круговая диаграмма с основными категориями.
2. **Экран "История (Лента)":**
    * Хронологический список транзакций с возможностью редактирования и удаления.
3. **Экран "Настройки/Управление":**
    * Управление списком категорий и счетов.

## 3. Визуальный язык и UI-кит (Linear Style)

Согласно современным стандартам и заложенным правилам дизайн-кода, приложение использует стиль **Linear Style** (Glassmorphism + Bento Grid).

* **Типографика:** `Inter` — чистый, отлично читаемый гротеск.
* **Цветовая палитра:**
  * **Фон (Background):** `#050505` (Глубокий черный, снижает напряжение глаз).
  * **Акцент (Primary):** `#6d5dfc` (Indigo — цвет кнопок, активных элементов).
  * **Деструктивный (Danger):** `#f43f5e` (Rose — мягкий красный для расходов и удалений).
  * **Успех (Success):** Мягкий зеленый (для доходов).
  * **Текст:** Основной — белый (`#ffffff`), вторичный — светло-серый (`rgba(255, 255, 255, 0.6)`).
* **Формы и эффекты (Glassmorphism):**
  * **Скругления:** `28px` для крупных карточек (виджетов Bento), `14px` для внутренних элементов (кнопки, плашки).
  * **Матовое стекло:** `backdrop-filter: blur(30px)` для модальных окон, нижнего меню (TabBar) и карточек-виджетов.
  * **Границы (Borders):** Тонкие, полупрозрачные `1px solid rgba(255, 255, 255, 0.08)` для создания объема без резкости.

## 4. Технический стек и Архитектура

* **Frontend Framework:** React + TypeScript (Vite).
* **Styling:** Vanilla CSS (CSS Variables) + Tailwind CSS (v4) для утилит.
* **Drag & Drop:** `@dnd-kit/core` + `@dnd-kit/sortable`. Использование `PointerSensor` с кастомными ограничениями (`delay: 200ms`, `tolerance: 15px`).
* **State Management:** React useState/useEffect (локальное хранилище `localStorage`).
* **Cloud Sync:** Google Apps Script (One-way Push via `POST text/plain`).
* **Иконки:** Lucide React.

## 5. Механики взаимодействия (UX)

Для минимизации когнитивной нагрузки и предотвращения случайных срабатываний реализована трехуровневая логика взаимодействия со счетами (кошельками):

1. **Редактирование (Long Press):** Удержание иконки на месте в течение 800 мс без значительного смещения открывает модальное окно настроек счета.
2. **Сортировка (Horizontal Drag):** Перетаскивание иконки влево или вправо внутри ряда счетов активирует `SortableContext`, позволяя менять порядок элементов.
3. **Действие - Трансфер/Расход (Vertical Pull-down):**
    * При смещении иконки вниз более чем на 35 пикселей, система переходит в «режим действия».
    * Визуальная сортировка (раздвигание соседних элементов) отключается, счета замирают и становятся статичными целями.
    * Наведение «монетки» на другой счет подсвечивает его фиолетовым и при отпускании открывает форму перевода денег между счетами.
    * Наведение на категорию в сетке ниже открывает форму записи расхода.

## 7. Новые возможности (Март 2026)

В марте 2026 года приложение получило ряд значительных обновлений, направленных на расширение функционала для путешественников и опытных пользователей.

### А. Мультивалютность и локализация

* **Поддержка валют:** В модель кошелька добавлено поле `currency`. Теперь каждый счет имеет свою «домашнюю» валюту (USD, RUB, BRL, EUR, ARS и др.).
* **Раздельный ввод сумм:** При переводах или расходах Numpad позволяет вводить две разные суммы (списания и получения). Это позволяет учитывать комиссии или реальный курс обмена.
* **Визуальные иконки:** Добавлены специальные значки валют для быстрой визуальной идентификации кошельков.
* **Полная русификация:** Интерфейс переведен на русский язык для снижения когнитивной нагрузки.

### Б. Встроенный калькулятор в Numpad

* **Математические операции:** Кнопки `+`, `-`, `*`, `/` позволяют проводить вычисления прямо в процессе ввода транзакции.
* **Проценты:** Поддержка оператора `%` (например, для быстрого расчета чаевых или налогов).
* **Safe Evaluation:** Реализована безопасная обработка выражений через `safeEval` с защитой от некорректного ввода.
* **Кнопка очистки (C):** Быстрый сброс текущего значения.

### В. Двусторонняя облачная синхронизация (Push & Pull)

* **Ручное обновление:** В шапку приложения добавлена кнопка «Обновить» (`RefreshCcw`), которая подтягивает актуальные настройки, кошельки и категории из Google Sheets.
* **Интеллектуальный маппинг:** Google Apps Script теперь динамически определяет индексы столбцов по их заголовкам. Это позволяет пользователю менять порядок столбцов в самой таблице без риска сломать синхронизацию.
* **Контроль версий:** Система отслеживает время последнего обновления (`cl_last_sync`) для предотвращения затирания актуальных данных старыми версиями.
