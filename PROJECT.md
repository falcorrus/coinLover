# CoinLover: Технический дизайн и концепция UI/UX (2025-2026)

## 1. Анализ рынка и тренды 2025-2026

Современные приложения для личных финансов отходят от образа сложных бухгалтерских таблиц в сторону «интеллектуальных финансовых партнеров».

**Ключевые тренды:**
* **Bento Grid Layout:** Модульная сетка для мгновенного считывания информации.
* **Linear Style & Glassmorphism:** Глубокие темные интерфейсы с эффектом матового стекла.
* **Micro-interactions:** Плавные анимации и мгновенный визуальный отклик.

## 2. Концепция CoinLover (MVP)

**Философия:** Только самое необходимое. Фокус на осознанности трат и простоте ввода.

### Основные сущности
1. **Счета (Кошельки):** Откуда уходят деньги (Наличные, Карта).
2. **Категории:** Куда уходят деньги (Продукты, Кафе).
3. **Транзакции:** Доходы, расходы, переводы.

## 3. Визуальный язык (Linear Style)

* **Типографика:** `Inter`.
* **Фон:** `#050505`. **Акцент:** `#6d5dfc`. **Успех:** Emerald. **Расход:** Rose/Gold.
* **Формы:** Скругления `28px` (карточки) и `14px` (кнопки). `backdrop-filter: blur(30px)`.

## 4. Технический стек

* **Frontend:** React + TypeScript (Vite).
* **Styling:** Vanilla CSS + Tailwind CSS (v4).
* **Drag & Drop:** `@dnd-kit/core`. Порог активации: `10px`.
* **Sync:** Google Apps Script (Интеллектуальный маппинг столбцов).

## 5. Механики взаимодействия (UX)

1. **Редактирование (Long Press):** Удержание 1.5 сек.
2. **Сортировка (Horizontal Drag):** Удержание 0.5 сек + движение в ряду.
3. **Действие (Quick Swipe):** Движение иконки вниз (>30px) отменяет режим сортировки и активирует транзакцию.

## 6. Новые возможности (Март 2026)

### А. Продвинутая мультивалютность (Source vs Target)
* **Разделение потоков:** Транзакции хранят `Source` (валюта кошелька) и `Target` (валюта операции). Это позволяет вводить расходы в местной валюте (BRL, THB) с любого кошелька.
* **Умный Numpad:** Двусторонняя синхронизация полей по курсу API. Возможность разрыва связи (Unlink) для ручного ввода суммы списания.
* **USD Analytics:** Автоматический расчет эквивалента в USD для всех типов операций.

### Б. Интеллектуальные индикаторы
* **Тройной переключатель (Pill):** Центральный виджет в шапке переключается между тремя режимами: «Расход за месяц», «Доход за месяц» и «Общий баланс».
* **Детализация доходов:** Аналитика доходов теперь показывает детализацию по **кошелькам**, в которые поступили деньги, с сохранением иконок и цветов.

## 7. Решение технических проблем (Knowledge Base)

Во время разработки мультивалютности были решены критические баги, знание которых необходимо для поддержки проекта:

### 1. Конфликт Скролла и Drag-and-Drop
**Проблема:** Иконки кошельков «дергались» или не захватывались при включенном `scroll-snap-type`.
**Решение:** Удален `scroll-snap` из CSS для горизонтальных списков. Браузерные механизмы «прилипания» конфликтуют с абсолютным позиционированием `@dnd-kit`. Оставлен только `-webkit-overflow-scrolling: touch` для инерции.

### 2. Ошибки инициализации состояний (ReferenceError)
**Проблема:** Приложение падало с ошибкой `confirmDelete is not defined`.
**Решение:** Внедрен стандарт «All States at Top». Все хуки `useState`, `useRef` и `useFinance` должны быть объявлены в самом верху компонента `App.tsx` до начала объявления функций-обработчиков.

### 3. Липкий кэш при смене БД (Demo/Real)
**Проблема:** При переключении с Demo на Real данные не обновлялись.
**Решение:** Функция `toggleMode` теперь принудительно вызывает `localStorage.removeItem()` для всех ключевых данных перед перезагрузкой. Хук `useFinance` настроен на `pullSettings()`, если локальный кэш пуст.

### 4. Порядок столбцов в Google Sheets
**Проблема:** Добавление новых полей в середину таблицы ломало старые скрипты.
**Решение:** В `Code.gs` реализован поиск индексов по названиям заголовков (`col["Source Amount"]`). Теперь порядок столбцов в таблице может быть любым.

### 5. Визуальная асимметрия Drag-and-Drop (Трансфер vs Расход)
**Проблема:** Перемещение между кошельками (Трансфер) и перемещение из кошелька в категорию (Расход) выглядели по-разному, хотя механика одна и та же. Категории имели градиентный фон, а кошельки — стиль «монетки».
**Решение:** Внедрен единый стандарт `.draggable-coin` для всех интерактивных элементов (Кошельки, Категории, Доходы). 
* Унифицирован эффект `coin-target-glow` (масштабирование 1.1x, внешнее свечение и замена цвета иконки на белый).
* Синхронизированы состояния `isPressing`, `isSortingMode` и `grabbed-elevation` во всех компонентах.
* Теперь интерфейс ощущается целостным: любое перетаскивание «монетки» на «монетку» визуально предсказуемо.
