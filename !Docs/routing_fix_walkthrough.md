# Результаты настройки веб-маршрутизации и изоляции PWA в CoinLover

Успешно решена проблема, при которой новые пользователи в браузере или при запуске в режиме standalone PWA (например, добавленный в Dock на macOS Sonoma или на домашний экран iOS/Android) ошибочно видели мобильный экран входа `NativeAuthScreen` вместо продающего лендинга.

## Внесенные изменения

### 1. Фронтенд (`src/App.tsx`)
* **Сужен критерий `isNativeApp`:** Теперь свойство `isNativeApp` возвращает `true` исключительно для настоящих нативных мобильных приложений, скомпилированных через Capacitor (`(window as any).Capacitor?.isNativePlatform`), а также при явной передаче отладочного флага `?force_native=true` в URL.
* **Изоляция PWA и Веба:** Все standalone-запуски через веб (standalone PWA, запущенные с Dock на Mac или с экрана "Домой" на iOS/Android) теперь классифицируются как веб-клиенты. 
* **Поведение маршрутизации:**
  * Если новый пользователь открывает `coinlover.ru` в обычном браузере или запускает свежеустановленное веб-PWA без `ssId` в памяти — ему гарантированно отображается **промо-лендинг `LandingPage`**.
  * Если пользователь запускает настоящее мобильное приложение (Capacitor iOS/Android) без подключенной таблицы — ему отображается премиальный мобильный вход **`NativeAuthScreen`**.
  * Если у пользователя в памяти браузера/PWA/приложения уже сохранен `ssId` таблицы — он мгновенно и бесшовно переходит в саму **программу** (прогу).
  * По адресу `coinlover.ru/landing` лендинг всегда открыт для всех.

## Как это работает сейчас (Итог для Браузера)
* **Для новых пользователей на `coinlover.ru` (в браузере и PWA):** Открывается лендинг.
* **Для существующих пользователей с `ssId` в памяти:** Сразу открывается прога.
* **Для принудительного просмотра лендинга:** Всегда работает адрес `coinlover.ru/landing`.
* **Для принудительного тестирования мобильного входа на вебе:** Работает адрес `coinlover.ru/?force_native=true`.
