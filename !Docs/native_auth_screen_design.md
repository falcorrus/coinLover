# Спецификация дизайна: NativeAuthScreen для CoinLover

Специализированный экран авторизации, оптимизированный для мобильных устройств, standalone PWA и Capacitor-приложений. Разработан в соответствии с визуальным стилем **Linear Style** и заменяет маркетинговый лендинг при запуске на мобильных платформах.

---

## 🎨 Визуальная идентичность (Design System)

Экран строго следует принципам **Linear Style** для создания премиального, «воздушного» ощущения от интерфейса:

- **Цветовая палитра:**
  - Глубокий темный фон: `#050505` (с мягкой фоновой сеткой `bg-white/5` с прозрачностью 20%).
  - Акцентный цвет: `#6d5dfc` (технологичный фиолетовый с тенью `shadow-[#6d5dfc]/20`).
  - Успешные состояния: Эмеральд `#10b981` (зеленый).
  - Ошибки: Rose/Red `#ef4444`.
- **Стекломорфизм (Glassmorphism):**
  - Использование контейнеров с эффектом матового стекла: `backdrop-blur-xl bg-white/[0.03] border border-white/10`.
  - Эффект глубокого размытия фонового градиента: радиальный меш `bg-[#6d5dfc]/8 blur-[100px]` позади карточки.
- **Типографика:**
  - Премиальный шрифт **Inter** / **Outfit** с жирными акцентными начертаниями для заголовков (`font-black tracking-tight`).
- **Скругления (Border Radius):**
  - Контейнер карточки авторизации и модальное окно сканера: `28px` (`rounded-[28px]`).
  - Текстовые поля, вкладки и основные кнопки действия: `14px` (`rounded-[14px]`).

## 📱 Оптимизация под Standalone & PWA (iOS / Android)

- **Динамический Safe Area Pt:**
  - При запуске в режиме standalone PWA на iOS Safari верхняя панель приложения наезжает на системную челку (notch). 
  - Реализован динамический расчет отступа сверху (`safePt`): при обнаружении standalone-режима автоматически вычисляется `calc(env(safe-area-inset-top, 24px) + 16px)`, что предотвращает наезд на элементы интерфейса и убирает дерганье сплеша.
- **Мгновенное определение PWA:**
  - Флаг `window.__IS_PWA__` считывается синхронно в `<head>` до инициализации React, помогая мгновенно переключить роутинг.
- **Камера в iOS WebKit:**
  - Для `<video>` элемента сканера QR-кода принудительно задан атрибут `playsinline="true"`, что предотвращает открытие нативного полноэкранного плеера iOS при запуске веб-камеры.

## 🛠 Функциональные сценарии авторизации

### 1. Основной сценарий: Telegram / Email (Решение 1)
- **Telegram вкладка:**
  - Текстовое поле ввода юзернейма. Автоматически очищает вводимое значение от символа `@` и переводит в нижний регистр.
  - Делает запрос к API `/api/sheets/` (метод `findUserByContact`), проверяет пользователя в мастер-таблице `Users` и возвращает его персональный `ssId`.
- **Email вкладка:**
  - Поле ввода электронной почты с валидацией формата.
  - Поиск `ssId` аналогично Telegram.
- **Успешный вход:**
  - Срабатывает легкое вибро-подтверждение (`navigator.vibrate(80)`).
  - `ssId` сохраняется в `localStorage` (`cl_active_table_id`) и `cookie` для предотвращения разлогина.
  - Отображается премиальный экран успеха с плавной анимацией лоадера, после чего происходит редирект.

### 2. Резервный сценарий: Для гиков (Решение 3)
- Плавно раскрывающийся спойлер **«Резервный вход (для гиков) 🛠»**.
- **Прямая ссылка:**
  - Текстовое поле ввода полной ссылки на Google Таблицу.
  - Регулярное выражение `/[-\w]{25,}/` мгновенно парсит Spreadsheet ID из ссылки.
  - Вызывает инициализацию таблиц `initTable(ssId)` через API.
- **Встроенный QR-сканер:**
  - Запускает камеру устройства и транслирует поток на скрытый `canvas`.
  - Кадры обрабатываются каждые 200 мс с помощью библиотеки `jsqr` полностью на клиенте (без внешних серверов).
  - При распознавании QR-кода с адресом таблицы извлекается `ssId`, срабатывает тройной вибро-отклик, и пользователь авторизуется.
- **Загрузка файла-изображения:**
  - Если доступ к камере запрещен или устройство не поддерживает захват видео, пользователь может нажать кнопку «Загрузить фото QR».
  - Картинка обрабатывается локально через холст и декодируется.

### 3. Быстрый демо-вход
- Ссылка «Попробовать Demo-режим» в подвале сбрасывает текущий `ssId` и запускает демо-режим, позволяя моментально ознакомиться с возможностями CoinLover.

## 🔍 Интеграция в архитектуру роутинга

В `src/App.tsx` внедрена проверка окружения:
```typescript
const isNativeApp = React.useMemo(() => {
  const params = new URLSearchParams(window.location.search);
  if (params.get("force_native") === "true") return true; // Дебаг-флаг для ПК

  return typeof window !== "undefined" && (
    (window as any).Capacitor?.isNativePlatform ||
    !!(window as any).__IS_PWA__ ||
    window.matchMedia('(display-mode: standalone)').matches ||
    (window.navigator as any).standalone
  );
}, []);
```

Если пользователь не авторизован и приложение открыто на мобильной платформе / PWA standalone, вместо сложного маркетингового лендинга скролла `LandingPage` мгновенно рендерится легкий, сфокусированный компонент `NativeAuthScreen`. Для тестирования на настольных ПК достаточно открыть приложение с параметром `?force_native=true`.
