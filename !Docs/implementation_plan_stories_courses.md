# Внедрение актуальных курсов валют в Сторис

Этот план описывает техническую реализацию наполнения карточки "Курсы" в разделе Историй (Stories) реальными данными о валютных курсах с использованием существующего `RatesService`, а также оптимизацию сортировки в календаре операций.

## Требования к логике курсов

1. **Вариант по умолчанию:** Курс рубля к базовой валюте пользователя (например, `USD/RUB`, если базовая валюта — USD). Если базовая валюта сама является `RUB`, по умолчанию показываем курс `USD/RUB` как наиболее востребованный.
2. **Мультивалютность кошельков:** Если у пользователя есть кошельки (счета) в других валютах (отличных от базовой), мы автоматически добавляем курсы этих валют к базовой.
3. **Красивое и стандартное представление:**
   - Все курсы валют приводятся к сравнению с базовой валютой пользователя в виде `BASE/TARGET` (например, `USD/ARS`, `USD/BRL`).
   - Для валют с очень малыми значениями (например, при базовой валюте `RUB` пара `RUB/USD` = `0.0113`) используется повышенная точность вывода (4 знака после запятой вместо стандартных 2), обеспечивая идеальную точность и удобство чтения.
4. **Реактивность и фоновое обновление:** При монтировании сторис запускается асинхронный метод `RatesService.ensureRates()`, который подтягивает самые свежие котировки с API. При успешном получении курсов интерфейс обновляется мгновенно без перезагрузок.
5. **Качественные фолбеки:** Если интернет отсутствует или API временно недоступно, приложение использует реалистичные базовые значения курсов с мягкой микродинамикой для идеального UX.

## Оптимизация Календаря операций

1. **Хронологический порядок дней:** Временная шкала дней (`timelineDays`) сохраняет стандартное возрастающее направление (от `1` до `31`), что обеспечивает естественное восприятие календаря.
2. **Свежие транзакции сверху:** Список операций за выбранный день (расположенный ниже временной полоски) сортируется по убыванию:
   - Сначала выводятся транзакции с более поздним временем.
   - Если транзакции имеют одинаковую дату (без времени), они сортируются по индексу в массиве в обратном порядке (т.е. самые недавно добавленные операции выводятся на самом верху списка).

## План верификации

### Автоматические проверки
- Убедиться, что проект компилируется без ошибок: `npm run build` или `npx tsc`.

### Ручное тестирование
1. Открыть Календарь операций.
2. Убедиться, что временная полоска дней идет в стандартном порядке (слева направо от 1-го числа к 31-му).
3. Добавить несколько транзакций в один день и открыть этот день в календаре: убедиться, что самая последняя (свежая) транзакция отображается на самом верху списка трат (ниже временной полоски).
