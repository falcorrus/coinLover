# 🔄 CoinLover: Синхронизация и Данные

## 🛰 Механизм VPS Прокси
- **Основной путь:** `src/services/universalFetch.ts` -> `api/sheets.ts`.
- **Обход CORS:** Использует нативный `CapacitorHttp` на мобильных устройствах и прокси Node.js в вебе.
- **Разрешение конфликтов:** 
    - Использует проверку `Timestamp` для сравнения локальной сессии и состояния облака.
    - Если данные в облаке новее, предлагает выбор: "Обновить локальные" или "Оставить мои".

## 📑 Парсинг и Маппинг
- **Гибкие заголовки:** Парсер игнорирует регистр и лишние пробелы.
- **Синонимы:** "Balance" = "Остаток" = "Сумма".
- **Мультивалютность:** Транзакции хранят `Source Amount` (кошелек) и `Target Amount` (операция) + `Source Currency` и `Target Currency`.

## 🔐 Безопасность
- **Passkeys:** Основной метод аутентификации. UID связан с контактом в Мастер-таблице.
- **Защита Мастер-таблицы:** Проверка подписки (колонка `Access Ends`) выполняется на бэкенде. Статус 403 вызывает блокирующую модалку в UI.
- **Сервисный аккаунт:** Пользователи должны предоставить доступ к своей таблице для `coinlover-service-acc@baonlineru.iam.gserviceaccount.com`.
