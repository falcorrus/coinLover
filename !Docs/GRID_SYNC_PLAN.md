# Идеальная Сетка CoinLover (Mockup & Logic)

## Проблема
Сейчас элементы имеют разную логику расчета ширины:
- **Кошельки/Доходы:** Фиксированные 80px + фиксированный gap.
- **Расходы:** `grid-cols-4` (процентная ширина) + фиксированный gap.
Это создает "лесенку" осей, которая усиливается к правому краю.

## Предлагаемое решение: Адаптивная Синхронизация

Мы привязываем количество колонок и их ширину к размеру экрана, сохраняя **минимум 80px**.

### 1. Малые экраны (до 380px, например iPhone SE)
- **Сетка:** 4 колонки.
- **Ширина элемента:** Ровно `(экран - отступы) / 4`. 
- **Результат:** Элементы ~80-84px. Идеальное совпадение осей.

### 2. Средние и Большие экраны (380px - 448px, iPhone Pro/Max)
- **Вариант А (Увеличение размера):** Сохраняем 4 колонки, но увеличиваем иконки и контейнеры (например, до 90-95px).
- **Вариант Б (Увеличение расстояния):** Сохраняем иконки 80px, но увеличиваем `gap` между ними, чтобы они равномерно распределились по ширине и совпали с сеткой расходов.

## Визуальный Мокап (ASCII)

### iPhone SE (Компактно)
```text
| px-4 [80] [80] [80] [80] px-4 |  <- Wallets
| px-4 [80] [80] [80] [80] px-4 |  <- Expenses
```
*Gap: 8px (фиксированный)*

### iPhone 15 Pro Max (Просторно)
```text
| px-6  [ 92px ] [ 92px ] [ 92px ] [ 92px ]  px-6 | <- Wallets
| px-6  [ 92px ] [ 92px ] [ 92px ] [ 92px ]  px-6 | <- Expenses
```
*Элементы "дышат", текст не режется, вертикальные оси строго совпадают.*

## Техническая реализация (План)

1. **Единая переменная ширины:** В `App.tsx` (или через CSS переменную) рассчитываем ширину колонки `column-width` на основе `window.innerWidth`.
2. **Accounts/Income:** Заменяем `w-[80px]` на `w-[var(--col-width)]`.
3. **Expenses:** Заменяем `grid-cols-4` на контейнер с `display: flex; gap: var(--gap);` где у каждого элемента `flex: 0 0 var(--col-width)`.

**Результат:** Перетаскивание монетки всегда идет по идеально ровной вертикальной линии, независимо от устройства.
