# Архитектура встроенного AI-аналитика в CoinLover

Альтернатива Telegram-боту — **встроенный AI-чат прямо внутри приложения CoinLover**. Это решение предоставляет бесшовный пользовательский опыт (UX), не требует интеграции с мессенджерами и позволяет пользователю делать аналитические запросы на естественном языке (например, *«сколько я потратил на еду в последние 50 дней»*) непосредственно в интерфейсе.

---

## 1. Как это работает (Пользовательский сценарий)
1. **Интерфейс:** В приложении появляется плавающая кнопка с иконкой Sparkles (ИИ) или вкладка «AI Аналитик».
2. **Чат-панель:** При клике открывается красивое окно чата (в фирменном Linear/Glassmorphism стиле: темный блюр-фон, фиолетовые акценты).
3. **Запрос:** Пользователь пишет: *«Сколько ушло на такси за последний месяц?»* или *«Какой мой средний расход в день в мае?»*.
4. **Обработка:** Приложение отправляет запрос на серверный эндпоинт, который подтягивает данные из Google Sheets пользователя, передает их в LLM (например, Gemini 1.5/3.5 Flash) и мгновенно выводит точный расчет и текстовый ответ.

---

## 2. Техническая архитектура решения

Компонентная схема взаимодействия выглядит следующим образом:

```
[ Frontend: Чат-панель ] 
       │
       ▼ (POST /api/ai-analyst)
[ API Endpoint (Vercel Serverless) ] 
       │
       ├─► [ Supabase / Auth ] ──► (Получает ID пользователя и его Sheet ID)
       │
       ├─► [ Google Sheets API ] ──► (Скачивает транзакции за последние 60-90 дней)
       │
       ▼ (Формирует компактный CSV-контекст)
[ LLM API (Gemini 1.5 Flash) ] ──► (Выполняет математический расчет и формулирует ответ)
       │
       ▼ (Возвращает Markdown-текст)
[ Frontend: Отображает ответ пользователю ]
```

---

## 3. Ключевые аспекты реализации

### А. Оптимальная передача данных в ИИ (Токены и Скорость)
Чтобы не переплачивать за API и не перегружать контекст модели, мы не передаем всю базу транзакций за много лет.
1. **Фильтрация по времени:** При получении запроса бэкенд выкачивает из Google Sheets транзакции только за разумный период (например, последние 60–90 дней). Этого достаточно для 95% аналитических вопросов пользователя.
2. **Компактный формат (CSV):** Вместо избыточного JSON транзакции форматируются в плоский CSV-текст, что экономит до 70% токенов:
   ```csv
   Дата,Категория,Сумма,Тип,Описание
   21.05.2026,Еда,1200,Расход,Супермаркет
   20.05.2026,Транспорт,450,Расход,Такси в офис
   18.05.2026,Зарплата,95000,Доход,Аванс
   ```

### Б. Разработка Системного Промпта (System Prompt)
Для получения точных и предсказуемых ответов модель инструктируется следующим образом:
```
Ты — профессиональный финансовый AI-аналитик CoinLover. 
Твоя задача — отвечать на вопросы пользователя на основе его личных транзакций.
Сегодняшняя дата: {CURRENT_DATE}.
Текущая валюта пользователя: {CURRENCY}.

Данные транзакций (CSV):
---
{CSV_DATA}
---

Правила:
1. Выполняй точные математические расчеты. Складывай суммы строго по выбранной категории и диапазону дат.
2. Если пользователь спрашивает про период времени, отсчитывай его назад от сегодняшней даты ({CURRENT_DATE}).
3. Отвечай кратко, профессионально, на русском языке. Форматируй важные цифры жирным шрифтом.
4. Если подходящих транзакций не найдено, вежливо сообщи: "За указанный период расходов в категории X не обнаружено".
```

### В. Выбор модели: Gemini 1.5 Flash / Gemini 3.5 Flash
> [!TIP]
> **Gemini Flash** — идеальный выбор для этой задачи:
> - **Огромное окно контекста** (до 1 млн токенов), что позволяет легко загружать тысячи строк транзакций при необходимости.
> - **Сверхвысокая скорость работы** (ответ генерируется за 0.5 - 1.2 секунды).
> - **Экстремально низкая стоимость** (в 10-15 раз дешевле Gemini Pro или Claude 3.5 Sonnet).
> - **Отличные математические способности** на структурированных CSV-данных.

## 4. Сложность реализации встроенного чата

*   **Общая сложность:** **Средняя** (3-5 дней работы для полной интеграции).
*   **Трудозатраты по слоям:**
    1.  **Frontend (UI):** 1-2 дня. Верстка панели чата, анимация появления, автопрокрутка вниз, индикатор загрузки (`typing...`), хранение истории текущей сессии в стейте.
    2.  **API эндпоинт (`/api/ai-analyst.ts`):** 1 день. Авторизация, получение личного `google_sheet_id`, вызов Google Sheets API для выгрузки транзакций, форматирование в CSV.
    3.  **Интеграция с LLM и промпт-инжиниринг:** 1 день. Интеграция SDK Google AI, настройка системного промпта, тестирование различных типов вопросов (граничные даты, опечатки в категориях, мультивалютность).

---

## 5. Преимущества встроенного чата перед Telegram-ботом

1.  **Отсутствие барьеров (Бесшовный UX):** Пользователю не нужно иметь Telegram, переходить по внешним ссылкам, проходить авторизацию (Deep Linking) или запускать сторонние приложения. Всё работает «из коробки».
2.  **Контекст авторизации:** Приложение на фронтенде уже авторизовано в Supabase, бэкенд автоматически знает `sheet_id` пользователя. Исключена любая возможность перепутать таблицы или получить чужие данные.
3.  **Единый дизайн-код:** Чат выглядит премиально и является органичной частью интерфейса CoinLover (соблюдая принципы Linear Style).
4.  **Безопасность:** Финансовые транзакции пользователя передаются напрямую по зашифрованному HTTPS-каналу между вашим бэкендом и API Google/Gemini, не оседая на серверах мессенджера.
