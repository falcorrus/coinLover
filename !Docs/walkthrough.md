# Внедрение Floating Action Button (FAB) для меню настроек и интеграция кастомной монетки SVG (Май 2026)

## Описание выполненных изменений

В соответствии с гайдлайнами Android (Material Design FAB) и для улучшения эргономики мобильного использования, меню настроек приложения CoinLover было перемещено из правого верхнего угла хедера в **правый нижний угол** в качестве плавающей кнопки действия (FAB). 

### 🌟 Ключевые изменения интерфейса
1. **Эргономика и доступность (Reachability)**:
   * На мобильных устройствах верхние углы экрана труднодоступны при одноручном использовании. Перенос меню в правый нижний угол в качестве FAB обеспечивает мгновенный доступ к основным действиям (темы, синхронизация, лента, аналитика) большим пальцем.
   * Кнопка позиционирована фиксированно: `fixed right-6 bottom-[calc(env(safe-area-inset-bottom,0px)+24px)] z-[150]`. Она учитывает отступ Safe Area на современных смартфонах с «челками» и нижними навигационными полосами (iOS/Android).
2. **Премиальный дизайн и иконка**:
   * Стандартная иконка бургер-меню ("три полоски") заменена на высококачественный **векторный логотип CoinLover** (инлайн-SVG на основе оригинального `favicon.svg`), представляющий собой золотую металлическую монетку с белым сердцем в центре.
   * Добавлены премиальные интерактивные эффекты:
     * Мягкий золотисто-фиолетовый градиент-блик при наведении (`group-hover`).
     * Плавная 3D-анимация вращения монетки на 360 градусов и легкое масштабирование при открытии меню (`rotate-[360deg] scale-90`).
3. **Обновление позиционирования меню**:
   * Само меню настроек теперь плавно всплывает **вверх над кнопкой FAB** (`bottom-[72px] right-0`), используя красивую выталкивающую анимацию (`animate-in fade-in slide-in-from-bottom-5 zoom-in-95 origin-bottom-right`) и закругления `rounded-[24px]` в стиле Glassmorphism.
   * Добавлен полупрозрачный бэкдроп-затемнитель (`bg-black/45 backdrop-blur-[2px]`) с z-индексом 140. Сама кнопка FAB и меню остаются полностью активными над затемнением благодаря z-индексам 150 и 145.
4. **Идеальный баланс и устранение дублирования хедера**:
   * Ранее при количестве категорий ≤ 8 в хедере возникало визуальное дублирование информации: текстовый заголовок `Total Balance` сверху хедера и интерактивная инфо-кнопка `PillButton` (баланс/расходы за месяц) на отдельной строке снизу.
   * Мы полностью удалили текстовый заголовок `Total Balance` и нижнюю строку.
   * Вместо этого интерактивная инфо-панель `PillButton` (выводящая расходы за месяц, доходы или общий баланс) была перемещена прямо на ее место — в самый верх, по центру хедера.
   * Справа по-прежнему сохраняется невидимая заглушка (`w-10 h-10` для компенсации кнопки FAB) для идеальной горизонтальной симметрии хедера. В результате шапка хедера сжалась до одной сверхкомпактной, информативной и визуально премиальной строки.

# Внедрение экрана авторизации NativeAuthScreen и оптимизация деплоя

## Описание выполненных изменений

Для мобильной версии приложения CoinLover (standalone PWA и Capacitor) успешно реализован премиальный, сфокусированный экран авторизации **NativeAuthScreen**. Он полностью заменяет сложный маркетинговый лендинг на мобильных устройствах, обеспечивая мгновенный, бесшовный и безопасный вход в приложение напрямую через привязку личной Google Таблицы.

### 1. Премиальный UI/UX (Linear Style)
* **Адаптивный экран авторизации:** Создан новый компонент `src/components/NativeAuthScreen.tsx`.
* **Стиль:** Дизайн выполнен в премиальном стиле Linear Style (глубокий черный фон `#050505`, акцентный фиолетовый `#6d5dfc`, HSL-палитра, скругления `28px` для окон и `14px` для полей ввода, glassmorphism-эффекты и тонкие интерактивные границы, плавные micro-анимации на базе `framer-motion`).
* **Безопасная зона (iOS Notch):** Добавлено автоматическое вычисление высоты верхней безопасной зоны (`safe-area-inset-top`) для standalone PWA, исключающее наложение элементов интерфейса на челку iPhone.

### 2. Единственный сценарий авторизации (По ссылке на таблицу)
* **Прямая вставка ссылки на Google Таблицу:**
  - На экране представлено единственное, лаконичное поле ввода для ссылки на личную Google Таблицу пользователя.
  - Клиентский код автоматически парсит Spreadsheet ID (`ssId`) из вставленной ссылки с помощью встроенного регулярного выражения `/[-_\w]{25,}/`.
  - При отправке формы приложение выполняет безопасный GET-запрос `googleSheetsService.fetchSettings(parsedSsId)` к нашему прокси-серверу для проверки доступности и валидации таблицы (это строго read-only операция, которая ничего не перезаписывает).
  - Если таблица доступна (пользователь предоставил доступ для сервисного аккаунта `coinlover-service-acc@baonlineru.iam.gserviceaccount.com`), приложение сохраняет `ssId` в `localStorage` и куки `cl_active_table_id` сроком на 1 год, дает мягкий вибро-отклик (`navigator.vibrate`) и перенаправляет пользователя на главный экран.
* **Вход в Demo-режим:**
  - Для ознакомления с возможностями приложения без привязки собственных таблиц предусмотрена кнопка «Попробовать Demo-режим», запускающая приложение на временных демо-данных.

### 3. Интеграция в приложение
* В `src/App.tsx` переписан механизм определения мобильного окружения. Приложение автоматически распознает запуск в Capacitor, standalone PWA на iOS или эмуляцию standalone в Android.
* Для удобства разработки добавлен GET-параметр `?force_native=true`, который принудительно открывает экран авторизации `NativeAuthScreen` даже в десктопном браузере (например, `http://localhost:5173/?force_native=true`).
* Селектор языков (RU/EN) полностью связан с глобальным состоянием, обеспечивая бесшовное переключение интерфейса авторизации.

### 4. Оптимизация и откат деплоя
* **Откат коммита:** Ранее в ветки `main`/`preview` по ошибке попали тяжелые Android-библиотеки и кэш-файлы (коммит `7094300`). 
* Мы провели принудительный сброс удаленного и локального репозиториев к стабильному коммиту `da9c76018bc6a71fc09ae873299990351085f806`.
* Все тяжелые файлы удалены, а гит-история приведена в полный порядок.
* Зависимость `jsqr` установлена с флагом `--ignore-scripts`, исключая сбои сборки native-модулей в Node.js на сервере.
* На сервере `server.reloto.ru` успешно запущены два изолированных докер-контейнера:
  - Продакшн: `coinlover` (порт `8010`, ветка `main`)
  - Дев-окружение: `dev-coinlover` (порт `8011`, ветка `preview`)

## План верификации

### Автоматические тесты
1. **Линтинг и типизация:** Запуск `npm run lint` (`tsc --noEmit`) проходит успешно без единой ошибки.
2. **Сборка проекта:** Команда `npm run build` успешно собирает продакшн-бандл через Vite.

### Ручное тестирование
1. **Тестирование авторизации (Google Sheet Link):**
   - Открыть `http://localhost:5173/?force_native=true` или перейти на дев-стенд `https://coin.reloto.ru/?force_native=true`.
   - Вставить корректную ссылку на Google Таблицу (на которую расшарен доступ для `coinlover-service-acc@baonlineru.iam.gserviceaccount.com`), нажать «Войти». Убедиться, что Spreadsheet ID парсится правильно, происходит вибрация и успешный переход в приложение.
   - Вставить неверную ссылку или ссылку на нерасшаренную таблицу. Убедиться, что выводится понятная ошибка валидации.
2. **Тестирование Demo-режима:**
   - Нажать кнопку «Попробовать Demo-режим». Убедиться, что открывается интерфейс CoinLover со всеми заполненными разделами в демонстрационном режиме.
3. **Проверка Safe Area:**
   - На мобильном iOS-устройстве в PWA-режиме (standalone) проверить, что шапка хедера и элементы `NativeAuthScreen` не залезают на системную челку iPhone и имеют сбалансированные отступы.
