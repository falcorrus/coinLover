# План реализации: Экран авторизации для нативного приложения CoinLover

В соответствии с требованиями для нативного приложения CoinLover (Capacitor/PWA Standalone) необходимо реализовать специализированный экран авторизации, который заменит маркетинговый лендинг. Это повысит конверсию, упростит первый запуск и сделает работу с приложением максимально комфортной.

---

## Что будет сделано

### 1. Добавление зависимости
- Установка библиотеки `jsqr` (`npm install jsqr`) для декодирования QR-кодов прямо из видеопотока или загруженного файла в браузере/WebView.

### 2. Экран авторизации (Новый компонент)
Создаем красивый, премиальный компонент `NativeAuthScreen.tsx` в стиле **Linear Style**:
- **Логотип и пульсация:** Красивый анимированный логотип CoinLover сверху.
- **Основной сценарий (Решение 1): Вход по Telegram / Email:**
  - Два переключаемых таба: **Telegram** и **Email**.
  - Для Telegram: инпут с префиксом `@`. Автоматически очищает вводимый юзернейм от лишних знаков.
  - Для Email: инпут с иконкой письма.
  - При отправке вызывается `googleSheetsService.findUserByContact(contact)`.
  - При успешном поиске: сохранение `ssId` в `localStorage` и `cookie`, показ вибро-отклика и переход в приложение.
- **Резервный сценарий (Решение 3): Для гиков (Вставка ссылки / QR-код):**
  - Кнопка-спойлер «Резервный вход (для гиков) 🛠», которая плавно раскрывает панель.
  - **Вставка ссылки:** текстовое поле для полной ссылки на Google Таблицу. Автоматически парсит `ssId` с помощью регулярного выражения.
  - **QR-код:** кнопка «Сканировать QR-код». Открывает аккуратное модальное окно с веб-камерой:
    - Рисуется рамка сканера с эффектом бегущей зеленой линии.
    - Кадры из видеопотока переносятся на невидимый холст (`canvas`) и обрабатываются через `jsQR`.
    - При распознавании URL таблицы извлекается `ssId`, вызывается вибросигнал и происходит вход.
    - Есть кнопка «Загрузить картинку» как фолбек, если камера недоступна или запрещены права.
- **Вход в Демо-режим:** Ссылка в самом низу «Попробовать Demo-режим» для быстрого ознакомления с приложением.

### 3. Интеграция в App.tsx
- Объединим определение нативного/standalone режима и будем показывать `<NativeAuthScreen />` вместо `<LandingPage />` на мобильных платформах.
