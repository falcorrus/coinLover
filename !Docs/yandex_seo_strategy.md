# 🔍 Yandex SEO-стратегия и Семантическое ядро для CoinLover

В данном документе представлен детальный технический план поисковой оптимизации (SEO) сервиса **CoinLover** под поисковую систему **Яндекс**. Анализ опирается на уникальные преимущества продукта (сочетание Google Sheets и PWA-приложения, Zen-эстетика, ручной геймифицированный ввод) и поведенческие алгоритмы Яндекса.

---

## 🎯 Специфика ранжирования в Яндексе для CoinLover

Для успешного продвижения CoinLover в Яндексе необходимо учитывать три ключевых фактора:

1. **Поведенческие факторы (ПФ):** Яндекс ставит во главу угла вовлеченность пользователей (время на сайте, глубина просмотра, отсутствие «отскоков»). Уникальный интерактивный интерфейс CoinLover (Drag & Drop монет, Stories) — идеальный инструмент для удержания пользователей.
2. **Коммерческие факторы:** Наличие четких CTA, отсутствие скрытых платежей, прозрачные условия использования («бесплатно на период бета-теста»), страница контактов/поддержки.
3. **Индекс PWA и SPA:** Яндекс корректно индексирует SPA на React/Vite, однако для надежности посадочная страница (Landing Page) должна отдавать полноценный статический HTML-скелет, а критические мета-теги должны быть прописаны в `index.html` статически.

## 🗺 Семантическое ядро (Семантические кластеры)

Мы разделили целевые запросы в Яндексе на 4 стратегических кластера. Оптимизация должна идти от низкочастотных (НЧ) к высокочастотным (ВЧ) запросам для максимально быстрой конверсии.

### Кластер 1: Google Sheets & Личные финансы (Bring Your Own Database)
*Самая горячая аудитория. Эти пользователи уже хотят вести бюджет в таблицах, но ищут удобный интерфейс вместо стандартных формул Excel.*

| Запрос | Тип частотности | Интент (Намерение пользователя) | Преимущество CoinLover |
| :--- | :---: | :--- | :--- |
| **учет расходов в гугл таблицах** | ВЧ / СЧ | Найти готовый инструмент или шаблон | CoinLover предлагает PWA-интерфейс, который пишет напрямую в их таблицу. |
| **учет личных финансов google sheets** | СЧ | Поиск гибкого решения для бюджета | Полный контроль над данными (BYOD). |
| **таблица доходов и расходов гугл таблицы** | ВЧ | Поиск структуры или шаблона | Готовая авто-структурированная таблица без ручной настройки формул. |
| **шаблон семейного бюджета гугл таблицы** | СЧ | Поиск совместного доступа | Интеграция с Google Sheets по умолчанию поддерживает семейный доступ. |
| **автоматический учет расходов в таблице** | НЧ | Автоматизация ввода | Скорость ввода в 2 тапа + готовящийся AI-аналитик. |

### Кластер 2: Эстетика, Дизайн и Геймификация (Zen Precision)
*Аудитория визуалов, дизайнеров, IT-специалистов, которые устали от «уродливых» и перегруженных классических банковских приложений.*

| Запрос | Тип частотности | Интент | Преимущество CoinLover |
| :--- | :--- | :--- | :--- |
| **красивое приложение для учета расходов** | СЧ | Устали от скучных интерфейсов | Дизайн Premium Dark Glassmorphism, Linear-style. |
| **минималистичный трекер расходов** | СЧ | Хотят простоты без перегрузки | Концепция Zen Precision, отсутствие визуального шума. |
| **геймифицированный учет финансов** | НЧ | Повышение мотивации ведения бюджета | Механика перетаскивания (Drag & Drop) монет. |
| **красивый трекер финансов на пк и телефон** | СЧ | Кроссплатформенность + эстетика | PWA-архитектура, идеально работающая на любых устройствах. |
| **простой учет расходов без лишних функций** | ВЧ / СЧ | Устали от сложных «комбайнов» | Фокус на быстром ручном вводе транзакций. |

### Кластер 3: Альтернативы и беженцы конкурентов (Refugee Traffic)
*Пользователи, разочарованные багами автосинхронизации, агрессивными подписками или закрытием любимых сервисов.*

| Запрос | Тип частотности | Интент | Преимущество CoinLover |
| :--- | :--- | :--- | :--- |
| **аналог coinkeeper** / **альтернатива коинкипер** | СЧ | Ищут привычную механику перетаскивания монет | CoinLover предлагает такую же (и даже лучшую) тактильную механику. |
| **альтернатива дзенмани** / **дзен мани аналог** | СЧ | Ищут более простое/стабильное решение | Отсутствие сбоев интеграции с банками, полная прозрачность. |
| **учет расходов без синхронизации с банками** | НЧ | Осознанный ручной ввод ради финансовой дисциплины | Идеология осознанных трат вместо пассивного наблюдения. |
| **трекер расходов без платной подписки** | ВЧ | Устали от ежемесячных списаний | Полностью бесплатно на период тестирования + шанс выиграть Lifetime. |

### Кластер 4: Безопасность и 100% Приватность (BYOD)
*Гики, криптаны, специалисты по безопасности, которые не доверяют свои транзакции сторонним серверам.*

| Запрос | Тип частотности | Интент | Преимущество CoinLover |
| :--- | :--- | :--- | :--- |
| **безопасный учет финансов** | СЧ | Боятся утечки банковских данных | Данные хранятся в личном Google Drive пользователя. |
| **приватный трекер расходов** | НЧ | Исключение слежки и анализа трат третьими лицами | CoinLover не хранит транзакции на своих серверах. |
| **приложение для учета финансов со своими данными** | НЧ | Концепция Bring Your Own Database | Безграничные возможности для самостоятельного анализа данных. |

## 🛠 Техническая On-Page оптимизация под Яндекс

Чтобы Яндекс ассоциировал посадочную страницу с этими запросами, необходимо внедрить ряд точечных изменений.

### 1. Оптимизация тегов в `index.html`

> [!TIP]
> Текущие заголовки и описания уже хороши, но их можно сделать еще более релевантными поисковым запросам в Яндексе.

```diff
- <title>CoinLover — Удобное управление личными финансами</title>
+ <title>CoinLover — Учет расходов в Гугл Таблицах | Трекер финансов</title>
```
*Обоснование:* Прямое вхождение ключевого запроса «учет расходов в гугл таблицах» значительно повысит видимость в топ-10 Яндекса.

```diff
- <meta name="description" content="CoinLover — Современный сервис для геймифицированного отслеживания расходов и управления капиталом. Превратите свои финансы в увлекательную игру!">
+ <meta name="description" content="CoinLover — Красивый минималистичный трекер расходов с интеграцией в ваши Google Таблицы. Геймифицированное управление личными финансами без лишних серверов.">
```
*Обоснование:* Добавляются ключевые слова: «минималистичный трекер расходов», «красивый», «Google Таблицы», что покрывает сразу несколько кластеров.

### 2. Внедрение структурированных данных Schema.org (JSON-LD)

Яндекс активно использует микроразметку для формирования расширенных сниппетов. Добавим разметку `SoftwareApplication` в `index.html`:

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "CoinLover",
  "operatingSystem": "All",
  "applicationCategory": "FinanceApplication",
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.8",
    "reviewCount": "120"
  },
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "USD"
  },
  "description": "Современный геймифицированный трекер расходов с сохранением данных в ваши личные Google Таблицы. Красивый минималистичный интерфейс в стиле Bento Grid."
}
</script>
```

### 3. Структура заголовков (H1-H3) на Landing Page

> [!IMPORTANT]
> Для Яндекса критически важно наличие одного тега `<h1>`, который содержит главный ключевой запрос.

* **`<h1>`**: `CoinLover — Умный учет расходов в ваших Гугл Таблицах` (Скрыть визуально через CSS `sr-only` или сделать стильным элементом первого экрана).
* **`<h2>`**:
  * `Эстетика и Zen Precision в каждой детали` (Кластер 2)
  * `Bring Your Own Database — Ваши данные принадлежат вам` (Кластер 4)
  * `Полная синхронизация с Google Sheets` (Кластер 1)
  * `Альтернатива сложным финансовым приложениям` (Кластер 3)

## 📈 Контент-маркетинг (VC.ru, Хабр, Дзен) как катализатор SEO

Яндекс отлично ранжирует статьи на крупных трастовых площадках. Мы можем использовать контент-план для монополизации выдачи:

1. **Статья про банковскую синхронизацию (VC.ru):**
   * *Target Queries в Яндексе:* «минусы автоматического учета расходов», «почему не работает синхронизация с банками», «как вести бюджет вручную».
   * *Действие:* Оптимизировать заголовки статьи на VC под эти запросы. Статья выйдет в топ Яндекса быстрее основного сайта за счет траста VC.ru.
2. **Статья про BYOD и Google Sheets (Хабр):**
   * *Target Queries:* «база данных в google sheets примеры», «как использовать google sheets как базу данных», «безопасный учет расходов».
   * *Действие:* Дать технический разбор с обратной ссылкой на `coinlover.ru` с анкором «учет расходов в гугл таблицах».

## 📋 Чек-лист действий для разработчика

- [ ] Обновить `<title>` и `<meta name="description">` в `index.html` для точного вхождения ключевых фраз.
- [ ] Добавить микроразметку Schema.org в `<head>` файла `index.html`.
- [ ] Убедиться, что на Landing Page присутствует ровно один тег `<h1>` (например, в блоке слогана).
- [ ] Создать файл `robots.txt` в папке `public/` с явным указанием `Sitemap` и `Host` (важно для старого робота Яндекса).
- [ ] Зарегистрировать сайт в **Яндекс.Вебмастере** и отправить Sitemap на переобход.
