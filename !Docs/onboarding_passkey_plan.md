# План реализации: Привязка Passkey при регистрации нового пользователя (Онбординг)

Этот план описывает реализацию бесшовной регистрации Passkey (биометрии/Face ID) на первом шаге онбординга нового пользователя, и последующую автоматическую привязку созданного ключа к Google Таблице после ввода ссылки на втором шаге.

---

## 📌 Архитектура нового флоу онбординга

```
[Шаг 1: Знакомство] ───> [Ввод Имени/Контакта] ───> [Клик "Далее"]
                                                         │
                                               (Если включен Passkey)
                                                         ▼
                                            [Нативная биометрия (Face ID)]
                                            - userID = contact
                                            - Создается пара ключей
                                            - Кред. сохраняется в памяти React
                                                         │
                                                         ▼
[Шаг 2: Таблица] ────> [Ввод ссылки Google Sheet] ────> [Клик "Подключить"]
                                                         │
                                                         ▼
                                            [Бэкенд: Регистрация Лида]
                                            - Сохранение в Master Sheet
                                            - Запись связки contact <-> ssId
                                                         │
                                                         ▼
                                            [Бэкенд: Регистрация Passkey]
                                            - Запись CredentialID и PublicKey
                                              в Configs пользователя
```

## 🛠 Предлагаемые изменения

### 1. Бэкенд (`api/auth.ts`)

#### А. Поддержка `contact` в `/api/auth/register-options`
Сейчас роут требует `ssId`. Мы добавим поддержку передачи `contact` вместо `ssId` для новых пользователей.
* Если передан `contact`:
  * Генерируем опции регистрации.
  * Используем `contact` в качестве `userID` (в виде Buffer).
  * `userName` и `userDisplayName` генерируем на основе `contact`.
  * Возвращаем challenge и `challengeToken` с закодированным в нем `contact` (вместо `ssId`).

#### Б. Поддержка связывания в `/api/auth/register-verify`
Когда пользователь вводит `ssId` на Шаге 2, клиент отправляет сохраненный в памяти WebAuthn-ответ и `ssId`.
* Бэкенд должен проверить challengeToken (в котором зашит `contact`).
* Валидирует подпись WebAuthn.
* После успешной валидации, подключается к Google Таблице по `ssId` и сохраняет ключи в лист `Configs` (`Passkey_Enabled = TRUE`, `Passkey_Credential_ID`, `Passkey_Public_Key`, `Passkey_Counter`).

#### В. Улучшенный поиск таблицы в `/api/auth/login-verify`
При входе по биометрии:
* Если в полученном от браузера `userHandle` содержится `contact` (а не `ssId`), бэкенд:
  1. Читает Master Sheet `Users` (из `MASTER_SS_ID`).
  2. Ищет строку, где колонка `Contact` совпадает с `contact` из `userHandle`.
  3. Извлекает соответствующий `ssId`.
  4. Если `ssId` найден, загружает публичный ключ из Google Таблицы пользователя по этому `ssId`.
  5. Проверяет подпись WebAuthn.
  6. При успешной авторизации возвращает `ssId` клиенту для входа.
* Если в `userHandle` содержится `ssId` (для старых пользователей, привязавших устройство в настройках):
  * Использует его напрямую без обращения к Master Sheet.

### 2. Фронтенд (`src/components/LandingPage.tsx`)

* **Состояние (State)**:
  * `pendingPasskeyCredential`: хранит WebAuthn-ответ (`RegistrationCredential` или `null`) после успешного прохождения биометрии на Шаге 1.
  * `pendingChallengeToken`: хранит `challengeToken` от сервера для последующей верификации.
* **Шаг 1 (Давайте знакомиться)**:
  * Показываем переключатель `Вход по отпечатку (Face ID)` (по умолчанию включен).
  * При клике на кнопку "Далее" (Next), если переключатель активен:
    * Меняем текст кнопки на *"Создание ключа доступа..."* и вызываем `navigator.credentials.create` через API `/api/auth/register-options?contact=...`.
    * При успехе: сохраняем результат в `pendingPasskeyCredential` и `pendingChallengeToken` и переходим на Шаг 2.
    * При ошибке биометрии (отмена пользователем, сбой Credential Manager): показываем мягкое уведомление и **беспрепятственно переходим на Шаг 2**, отключив флаг `usePasskeyForOnboarding`. Пользователь не должен блокироваться!
* **Шаг 2 (Подключение таблицы)**:
  * Пользователь вводит ссылку на таблицу и нажимает "Подключить".
  * Отправляем `registerLead` на бэкенд.
  * Если `pendingPasskeyCredential` есть:
    * Отправляем запрос на `/api/auth/register-verify` с `ssId = parsedSsId`, `registrationResponse = pendingPasskeyCredential`, `challengeToken = pendingChallengeToken`.
    * После верификации сохраняем сессию в `localStorage` и делаем редирект.
    * Если произошла ошибка привязки биометрии на этом этапе — показываем красивый экран ошибки биометрии с кнопкой **"Продолжить без биометрии"**, которая позволяет сразу войти в приложение по обычной ссылке.

## 🧪 План верификации

### Ручные сценарии:
1. **Сценарий 1: Успешный онбординг с Passkey**
   * Открываем модалку регистрации, пишем имя и контакт.
   * Нажимаем "Далее". Появляется системный диалог Face ID/отпечатка. Проходим его успешно.
   * Переходим на Шаг 2. Вставляем ссылку на созданную таблицу. Нажимаем "Подключить".
   * Система регистрирует лид, затем привязывает ключ к таблице.
   * Успешный вход в приложение. Проверяем в Google Sheets, что в `Configs` записались ключи Passkey.
2. **Сценарий 2: Отмена/ошибка биометрии на Шаге 1**
   * Вводим имя и контакт, нажимаем "Далее".
   * В системном диалоге биометрии нажимаем "Отмена".
   * Модалка плавно переходит на Шаг 2 с сообщением, что биометрия будет отключена.
   * Подключаем таблицу — завершаем регистрацию без ошибок.
3. **Сценарий 3: Вход по Passkey**
   * Выходим из приложения (очищаем куки/localStorage).
   * Нажимаем "Войти через Face ID / Passkey".
   * Прикладываем палец. Приложение успешно декодирует контакт, находит `ssId` в Master Sheet и мгновенно впускает в программу!
