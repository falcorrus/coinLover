# 🛠 CoinLover: Разработка и Процессы

## 👨‍💻 Инженерные стандарты
- **Karpathy Style:** Простота важнее абстракции. Явное лучше неявного.
- **Безопасный стейт:** Все хуки `useState`/`useRef` должны быть в верхней части компонентов (особенно в `App.tsx`), чтобы избежать `ReferenceError`.
- **Безопасность координат:** `DndContext` и `DragOverlay` должны оставаться на корневом уровне, чтобы избежать проблем со смещением на широких экранах.

## 🧪 Тестирование и Валидация
- **Локальное тестирование:** `pnpm run dev` для веба, `npx cap open android` для мобильных устройств.
- **Проверка сборки:** Всегда запускайте `pnpm run lint` и `pnpm run build` перед деплоем.
- **Обновление APK:** Используйте `./build_and_send.sh` для создания свежего билда и отправки его в Telegram. **Вручную синхронизируйте** `apk/coinlover-debug.apk` в `public/download/coinlover.apk` для лендинга.

## 📝 Цикл успеха (Success Loop)
Когда задача успешно завершена:
1. **Анализ:** Зафиксируйте техническую суть исправления.
2. **SOLUTIONS.md:** Обновите `!Docs/!SOLUTIONS.md`.
3. **Память:** Обновите `memoir` и глобальную память через `manager.py`.
