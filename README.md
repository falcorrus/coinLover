**Prerequisites:**  Node.js

   `npm install`
2. Set the `GEMINI_API_KEY` in [.env.local](.env.local) to your Gemini API key
3. Run the app:
   `npm run dev`

# О проекте

  CoinLover — это современное приложение для учета личных финансов с инновационным интерфейсом Drag & Drop в стиле Glassmorphism.

### Ключевые возможности

- **Умное перетаскивание:** Система автоматически различает сортировку счетов (движение вбок) и финансовые операции (движение вниз).
- **Темы оформления (Midnight Gold):** Внедрена третья тема в стиле AliGrids (Midnight Blue + Liquid Amber) с акцентом на тактильность и глубину.
- **Мгновенное переключение тем:** Панель выбора из трех иконок (Sun, Moon, Sparkles) в верхней части настроек для смены стиля в один клик.
- **Трансферы:** Мгновенный перевод между кошельками простым перетаскиванием одного на другой.
- **Интерактивная аналитика:** Выбор конкретных категорий и тегов для формирования итогов. Выбор сохраняется в течение дня.
- **Калькулятор (Numpad):** Встроенный калькулятор с поддержкой тегов и быстрым выбором даты (Сегодня/Вчера).
- **Лента транзакций:** Улучшенный интерфейс сгруппированной истории с выделением тегов на второй строке.

### Управление кошельками

  1. **Клик/Удержание (0.8с):** Редактирование счета.
  2. **Сдвиг в сторону:** Изменение порядка кошельков (сортировка).
  3. **Сдвиг вниз:** Перевод другому кошельку или расход в категорию.

### Technical Implementation: Cloud & Sync

  Для надежной работы на нескольких устройствах реализована следующая логика:

  1. **Атомарные снимки (Snapshots):**
     - При каждой транзакции приложение отправляет в гугл-скрипт не только саму операцию, но и полный снимок текущего состояния всех балансов, категорий и доходов.
     - `Code.gs` был оптимизирован для пакетной записи (`setValues`), что исключает частичное сохранение данных.
  2. **Background Conflict Detection:**
     - При старте `App.tsx` выполняет фоновый запрос к БД.
     - Если временная метка (Atomic Snapshot Timestamp) в облаке отличается от последней синхронизированной версии на устройстве, открывается **Conflict Resolution Modal**.
  3. **Разрешение «в один клик»:**
     - Пользователь может либо «Накатить» данные из облака, либо «Перезаписать» облако своей локальной версией.
  4. **Автоматическое резервное копирование (Auto-Backups):**
     - Вместо нестабильных Cron-триггеров в Apps Script внедрен внутренний счетчик через `PropertiesService`.
     - Ровно на каждую 10-ую успешную синхронизацию настроек (`syncSettings`), БД делает полную копию листа `Configs` (включая цвета и ширину колонок) в лист-архив `ConfigsArch`.
  5. **Правильная часовая зона:**
     - Приложение передаёт в БД не UTC-время (`toISOString()`), а отформатированное локальное системное время пользователя на момент совершения транзакции (`getLocalTimeString()`), что решает проблему смещенных таймстепмов в Google Таблицах.

### Technical Implementation: Precise DnD

  Чтобы добиться идеального попадания и исключить случайные захваты, мы внедрили следующие решения:

  1. **Разделение зон захвата (Icon-Only Grab):**
     - Обработчики перетаскивания (`listeners`) установлены **только на иконку (круг)**.
     - Текстовые подписи и пространство между иконками свободны от DnD-событий, что позволяет пользователю беспрепятственно использовать горизонтальный скролл.
  2. **Трехступенчатая логика масштабирования (Scale Logic):**
     - **Нажатие (Touch):** `scale(0.9)` — монетка визуально «сжимается» для подтверждения касания.
     - **Захват/Сортировка:** `scale(1.1)` — монетка «всплывает» и получает эффект свечения.
     - **Покой:** `scale(1.0)`.
  3. **Кастомная детекция столкновений (Collision Detection):**
     - В `DndContext` используется фильтрация `pointerWithin`. Столкновение засчитывается только если координаты указателя находятся непосредственно над иконкой-целью. Это делает попадание «пиксель-в-пиксель».
  4. **Логика удержания и тайминги:**
     - **600мс:** Переход в режим сортировки (`onSortingMode`).
     - **2000мс:** Открытие модального окна редактирования (`onLongPress`).
     - **15px:** Порог движения, после которого клик отменяется и начинается Drag (транзакция).
  5. **Единый визуальный стандарт:**
     - И кошельки, и категории используют общий класс `.draggable-coin` и `.coin-target-glow`, что обеспечивает визуальную симметрию трансферов и расходов.

- Двусторонняя интеграция с Google Sheets (Push + Pull Sync).
- Система разрешения конфликтов (Conflict Modal).
- Поддержка доходов (Emerald #10B981), расходов (Golden #D4AF37) и бюджетных категорий.
- Адаптивный дизайн (Bento Grid) для мобильных устройств.
