**Prerequisites:**  Node.js

   `npm install`
2. Set the `GEMINI_API_KEY` in [.env.local](.env.local) to your Gemini API key
3. Run the app:
   `npm run dev`

# О проекте

  CoinLover — это современное приложение для учета личных финансов с инновационным интерфейсом Drag & Drop в стиле Glassmorphism.

### Ключевые возможности

- **Умное перетаскивание:** Система автоматически различает сортировку счетов (движение вбок) и финансовые операции (движение вниз).
- **Трансферы:** Мгновенный перевод между кошельками простым перетаскиванием одного на другой.
- **Калькулятор (Numpad):** Встроенный калькулятор с поддержкой тегов и быстрым выбором даты (Сегодня/Вчера).
- **Облачная синхронизация (Cloud Sync):** Автоматическая отправка данных в Google Таблицы и интеллектуальное обнаружение конфликтов.
- **Разрешение конфликтов:** При открытии на другом устройстве приложение предложит выбрать между локальными данными и версией в облаке.
- **Persistence:** Гибридное хранение — `localStorage` для скорости и Google Sheets для отказоустойчивости.

### Управление кошельками

  1. **Клик/Удержание (0.8с):** Редактирование счета.
  2. **Сдвиг в сторону:** Изменение порядка кошельков (сортировка).
  3. **Сдвиг вниз:** Перевод другому кошельку или расход в категорию.

### Technical Implementation: Cloud & Sync

  Для надежной работы на нескольких устройствах реализована следующая логика:

  1. **Атомарные снимки (Snapshots):**
     - При каждой транзакции приложение отправляет в гугл-скрипт не только саму операцию, но и полный снимок текущего состояния всех балансов, категорий и доходов.
     - `Code.gs` был оптимизирован для пакетной записи (`setValues`), что исключает частичное сохранение данных.
  2. **Background Conflict Detection:**
     - При старте `App.tsx` выполняет фоновый запрос к БД.
     - Если временная метка (Atomic Snapshot Timestamp) в облаке отличается от последней синхронизированной версии на устройстве, открывается **Conflict Resolution Modal**.
  3. **Разрешение «в один клик»:**
     - Пользователь может либо «Накатить» данные из облака, либо «Перезаписать» облако своей локальной версией.

### Technical Implementation: Precise DnD

  Чтобы добиться идеального попадания и исключить смещение (offset), мы внедрили следующие решения:

  1. **Разделение Refs:**
     - `setSortRef` (из `useSortable`) установлен на **родительский контейнер** всего элемента (иконка + текст), чтобы при сортировке двигался весь блок.
     - `setDropRef` (из `useDroppable`) установлен **только на иконку (круг)**, чтобы логическая область попадания при транзакции совпадала с визуальной.
  2. **Устранение "прыжка" (Visual Lift):**
     - В CSS классе `.grabbed-elevation` удален `translateY(-10px)`. Теперь иконка при захвате только увеличивается (`scale`), оставаясь точно под пальцем/курсором.
  3. **Кастомная детекция столкновений (Collision Detection):**
     - В `DndContext` (App.tsx) используется фильтрация `pointerWithin`. Столкновение засчитывается только если координаты указателя находятся в верхних 70px целевого прямоугольника (зона иконки). Это игнорирует текстовые подписи снизу и делает попадание "пиксель-в-пиксель".
  4. **Логика удержания (Long Press):**
     - Внутри каждого `AccountItem/CategoryItem` работают таймеры в `onPointerDown`:
       - **500ms:** Переход в режим сортировки (`onSortingMode`).
       - **1500ms:** Открытие модального окна редактирования (`onLongPress`).
     - Если палец сдвинулся более чем на 10px до срабатывания таймеров, они сбрасываются, и обычный Drag (транзакция) начинается немедленно.

  Текущее состояние проекта:

- Полностью рабочий механизм DnD на базе `@dnd-kit`.
- Двусторонняя интеграция с Google Sheets (Push + Pull Sync).
- Система разрешения конфликтов (Conflict Modal).
- Поддержка доходов (Emerald #10B981), расходов (Golden #D4AF37) и бюджетных категорий.
- Адаптивный дизайн (Bento Grid) для мобильных устройств.
